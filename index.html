Sovereign AI Agents
with Persistent Memory

Tylluan is an open-source agent framework that remembers. A Rust kernel orchestrates specialized Python guilds — web search, code execution, memory — connected via MCP. Runs 100% locally. Built for researchers and builders who need agents that don't forget.

What Tylluan does

Agents that remember
what you taught them

Unlike stateless chat interfaces, Tylluan agents accumulate knowledge over time using embeddings and a knowledge graph. They're specialized, observable, and run entirely on your hardware.

🧠

Persistent Memory

Memories stored via BGE-M3 embeddings + graph traversal (SilvaDB). Recall what the agent learned last month. No RAG pipeline to manage.

⚙️

34 Specialized Guilds

Python subprocesses for web research, code execution, file operations, embedding generation, inter-agent messaging, and more. Each isolated.

🔌

MCP Compatible

Exposes sovereign tools via Model Context Protocol — SSE and HTTP Streamable. Connect Claude Desktop, Claude Code, Cursor, or any MCP client.

🦉

Sovereign Tools

tylluan_do (execute), tylluan_remember (store), tylluan_recall (retrieve), tylluan_think (analyze), tylluan_graph (navigate). Five primitives, infinitely composable.

🔒

Security by Design

Localhost-only by default. Bearer token auth. Audit logging. OWASP ASI01–ASI10 threat model documented. No cloud dependencies in the critical path.

📊

Observable

Every operation logged to audit.db. Graph stats via API. Health endpoint. The dashboard shows guild status and memory topology.

Security model

Honest about the risks.
Transparent about the gaps.

Tylluan is an experimental lab framework. It executes real code on your machine. We document every risk — we don't hide them behind marketing.

⚠ Experimental Lab Software

Tylluan is not a product. It is a research framework that executes real shell commands on your machine. Misconfiguration can expose your system. The operator — you — bears final responsibility for security. Always review guild code before enabling it. Never expose Tylluan to untrusted networks. See DISCLAIMER.md and Security Model for the full picture.

✅ Localhost-only by default

Binds to 127.0.0.1. No network exposure without deliberate config change.

✅ Bearer token authentication

MCP endpoint requires auth token. Configurable, rotatable.

✅ Guild subprocess isolation

Each guild runs in its own Python subprocess. Failure doesn't corrupt the kernel.

✅ Audit logging

Every operation recorded in audit.db with timestamp, tool, guild, and author_id.

⚠ No kill switch (yet)

If an agent misbehaves, manual taskkill required. Roadmap item.

⚠ No code execution sandbox

Bash guild and code guild run with operator's full permissions. Sandbox planned.

⚠ No encryption at rest

audit.db, memories, and inter-agent messages stored in plaintext.

⚠ author_id is self-reported

No cryptographic verification of agent identity. Spoofable with bearer token.

Get started

Zero to hello world
in 15 minutes

Prerequisites: Rust 1.75+, Python 3.11+, Git. Everything else is included. 

# 1. Clone the clean release branch
git clone https://github.com/tylluan/tylluan.git
cd tylluan

# 2. Build the kernel
cargo build -p tylluan-kernel

# 3. Configure (change bearer_token!)
cp tylluan.example.toml tylluan.toml
$EDITOR tylluan.toml

# 4. Start Tylluan
./tylluan-mcp.sh   # Linux/Mac
.\tylluan-mcp.bat   # Windows

# 5. Verify it's running
curl http://127.0.0.1:3030/health
# {"status":"ok","version":"1.0.0"}

# 6. Connect your MCP client (Claude Desktop example)
# Add to ~/.config/claude/claude_desktop_config.json:
{
  "mcpServers": {
    "tylluan": {
      "url": "http://127.0.0.1:3030/sse"
    }
  }
}

# 7. First command via MCP client:
Use tylluan_remember that Tylluan is my agent framework
Use tylluan_recall to find memories about Tylluan
